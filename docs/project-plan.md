# 校园生活百事通项目规划

## 1. 项目概述

校园生活百事通是一个面向高校学生的智能问答知识库系统。系统集中管理校园生活信息，为学生提供自然语言问答、热点问题查看、账户登录、个人中心和新问答贡献能力。管理员功能独立到 `admin-console` 管理端，用于知识维护、批量导入和投稿审核。

## 2. 项目目标

总体目标是完成一个可演示、可部署、可扩展的校园生活智能问答系统。

阶段目标：

1. 完成知识库管理能力。
2. 完成自然语言问答能力。
3. 完成学生注册、登录和个人中心能力。
4. 完成用户贡献和管理员审核能力。
5. 完成热点问题统计和展示能力。
6. 完成课程答辩所需演示数据和运行说明。

## 3. 项目范围

第一版范围：

1. 学生桌面客户端。
2. `admin-console` 管理端。
3. 云端 API 服务。
4. 知识库维护。
5. FAQ 批量导入。
6. 智能检索问答。
7. 学生注册和登录。
8. 个人中心和投稿历史。
9. 用户投稿。
10. 管理员审核。
11. 热点问题排行榜。

第一版不包含：

1. 多学校多租户。
2. 移动端 App。
3. 公众号自动抓取。
4. 复杂官网爬虫。
5. 人工客服转接。
6. 大规模分布式部署。

## 4. 项目角色

| 角色 | 主要职责 |
|---|---|
| 学生用户 | 注册登录、提问、查看热点问题、提交新问答、查看投稿历史 |
| 管理员 | 通过 `admin-console` 维护知识库、导入 FAQ、审核投稿 |
| 系统服务 | 执行问答检索、记录日志、统计热点 |

## 5. 阶段计划

### 5.1 阶段一：基础框架

目标：完成学生客户端、`admin-console`、云端服务和数据库的基础可运行环境。

主要任务：

1. 初始化项目结构。
2. 搭建学生客户端基础页面。
3. 搭建 `admin-console` 基础页面。
4. 搭建云端服务框架。
5. 准备数据库运行环境。
6. 建立配置和日志基础能力。

交付物：

1. 可启动的学生客户端。
2. 可启动的 `admin-console`。
3. 可启动的云端服务。
4. 基础运行说明。

### 5.2 阶段二：知识库管理

目标：完成管理员维护知识库的基础能力。

主要任务：

1. 管理员登录。
2. 知识新增、编辑、删除、查询。
3. 分类和标签维护。
4. FAQ 批量导入。

交付物：

1. `admin-console` 登录页面。
2. `admin-console` 知识库管理页面。
3. FAQ 导入功能。

### 5.3 阶段三：问答检索

目标：完成学生自然语言提问和系统智能检索。

主要任务：

1. 学生问答页面。
2. 问题预处理。
3. 意图识别。
4. 关键词检索。
5. 语义检索。
6. 混合排序。
7. 未命中提示。

交付物：

1. 可用的问答入口。
2. 可演示的智能检索效果。
3. 示例知识库数据。

### 5.4 阶段四：学生账户与个人中心

目标：完成学生注册、登录和个人中心能力。

主要任务：

1. 学生注册。
2. 学生登录。
3. 学生登录状态维护。
4. 个人中心展示基础账户信息。
5. 个人中心展示自己的投稿历史。

交付物：

1. 学生注册页面。
2. 学生登录页面。
3. 个人中心页面。
4. 投稿历史列表。

### 5.5 阶段五：用户贡献与审核

目标：完成知识库众包更新闭环。

主要任务：

1. 已登录学生提交新问答。
2. 管理员查看投稿。
3. 管理员审核通过或驳回。
4. 审核通过后进入知识库。

交付物：

1. 学生端用户贡献页面。
2. `admin-console` 投稿审核页面。
3. 投稿进入知识库流程。

### 5.6 阶段六：热点问题

目标：完成高频查询问题统计和展示。

主要任务：

1. 记录查询行为。
2. 统计热点问题。
3. 展示热点排行榜。

交付物：

1. 热点问题页面。
2. 热点问题统计结果。

### 5.7 阶段七：收尾与答辩

目标：完善系统演示效果和文档材料。

主要任务：

1. 补充演示数据。
2. 优化页面交互。
3. 补充异常提示。
4. 编写运行说明。
5. 准备答辩演示流程。

交付物：

1. 演示数据集。
2. 项目运行说明。
3. 答辩演示脚本。

## 6. 里程碑

| 里程碑 | 内容 | 验收标准 |
|---|---|---|
| M1 | 基础框架完成 | 学生客户端、`admin-console` 和云端服务可启动 |
| M2 | 知识库管理完成 | 管理员可在 `admin-console` 维护知识 |
| M3 | 问答检索完成 | 学生提问后可获得答案 |
| M4 | 学生账户完成 | 学生可注册、登录并查看个人中心 |
| M5 | 投稿审核完成 | 学生投稿可审核入库 |
| M6 | 热点统计完成 | 可展示热点问题排行榜 |
| M7 | 项目收尾完成 | 可完整演示核心流程 |

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 知识数据不足 | 演示效果弱 | 提前准备覆盖多个校园场景的示例数据 |
| 检索效果不稳定 | 回答命中率低 | 先完成关键词检索，再逐步增强语义检索 |
| 外部模型不可用 | 智能增强失败 | 保证基础问答不依赖外部模型 |
| 文件导入格式混乱 | 导入失败率高 | 第一版限制导入模板格式 |
| 联调时间不足 | 影响整体演示 | 先用接口工具验证云端，再分别接学生客户端和 `admin-console` |

## 8. 验收标准

项目验收时应满足：

1. 学生客户端可以正常启动。
2. `admin-console` 可以正常启动。
3. 云端服务可以正常启动。
4. 管理员可以在 `admin-console` 维护知识库。
5. 管理员可以批量导入 FAQ。
6. 学生可以自然语言提问并获得答案。
7. 学生可以注册和登录。
8. 学生可以在个人中心查看自己的投稿历史。
9. 已登录学生可以提交新问答。
10. 管理员可以审核用户投稿。
11. 系统可以展示热点问题排行榜。
12. 系统具备完整演示数据和运行说明。
