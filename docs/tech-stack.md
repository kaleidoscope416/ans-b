# 校园生活百事通技术栈方案

## 1. 技术栈目标

本项目技术栈以课程设计可落地、部署简单、演示稳定、后续可扩展为原则。系统拆分为学生桌面客户端、`admin-console` 管理端和云端服务。学生客户端负责问答和投稿，`admin-console` 负责管理员操作，云端服务负责业务处理和智能问答。

## 2. 学生客户端技术栈

| 类型 | 选型 | 用途 |
|---|---|---|
| 桌面框架 | Wails | 构建跨平台桌面客户端 |
| 前端框架 | Vue 3 | 实现学生端页面和交互 |
| UI 组件库 | TDesign Vue Next | 提供问答、热点和投稿页面组件 |
| 状态管理 | Pinia | 管理学生登录状态、页面状态和问答结果 |
| HTTP 请求 | Axios | 调用云端 REST API |
| 构建工具 | Vite | 前端开发和构建 |

选择理由：

1. Wails 可以使用 Go 构建桌面应用，与后端语言统一。
2. Vue 3 学习成本低，适合快速实现课程项目页面。
3. TDesign Vue Next 组件完整，适合快速实现表单和结果展示。
4. Axios 和 Pinia 生态成熟，便于维护客户端状态和接口调用。

## 3. admin-console 技术栈

`admin-console` 是独立管理端，用于管理员维护知识库、导入 FAQ 和审核投稿。

| 类型 | 选型 | 用途 |
|---|---|---|
| 前端框架 | Vue 3 | 实现管理端页面 |
| UI 组件库 | TDesign Vue Next | 提供表格、表单、弹窗、分页和上传组件 |
| 状态管理 | Pinia | 管理管理员登录状态和页面状态 |
| HTTP 请求 | Axios | 调用管理员 REST API |
| 构建工具 | Vite | 管理端开发和构建 |

选择理由：

1. 管理员功能和学生功能分离，界面职责更清晰。
2. 管理端以表格、表单和审核流程为主，适合使用 TDesign Vue Next。
3. 后续可以将 `admin-console` 独立部署到服务器或内网环境。

## 4. 云端技术栈

| 类型 | 选型 | 用途 |
|---|---|---|
| 编程语言 | Go | 编写云端业务服务 |
| Web 框架 | Gin | 提供 REST API |
| ORM | GORM | 简化数据库访问 |
| 配置管理 | Viper | 管理服务端口、数据库连接、模型配置 |
| 日志 | slog 或 zap | 记录运行日志和错误日志 |
| 鉴权 | JWT + Redis | JWT 作为客户端凭证，Redis 存储服务端登录状态 |
| 密码处理 | bcrypt | 学生和管理员密码哈希存储 |

选择理由：

1. Go 部署简单，适合作为单体 API 服务。
2. Gin 轻量成熟，适合快速开发 REST 接口。
3. GORM 可以降低数据库 CRUD 实现成本。
4. JWT 便于客户端携带凭证，Redis 支持服务端主动失效登录状态。

## 5. 数据与检索技术栈

| 类型 | 选型 | 用途 |
|---|---|---|
| 主数据库 | PostgreSQL | 保存业务数据 |
| 向量扩展 | pgvector | 保存文本向量并执行语义检索 |
| 关键词检索 | PostgreSQL 全文检索或 LIKE | 实现基础关键词匹配 |
| embedding | 外部 API 或本地模型 | 将问题和知识内容转换为向量 |
| 大模型 | 可选外部 API 或本地模型 | 根据检索结果生成更自然的回答 |

选择理由：

1. PostgreSQL 同时支持结构化数据和扩展能力。
2. pgvector 能满足第一版语义检索需求，避免额外部署向量数据库。
3. 大模型作为可选增强，不影响基础问答闭环。

## 6. 文件与导入技术

| 类型 | 选型 | 用途 |
|---|---|---|
| 第一版文件存储 | 本地文件系统 | 保存上传文件和导入文件 |
| 后续文件存储 | MinIO | 支持对象存储扩展 |
| 表格导入 | CSV / Excel 解析库 | 批量导入 FAQ |
| 文档解析 | 后续扩展 PDF / Word / Markdown 解析 | 支持更多知识来源 |

第一版优先支持 FAQ 表格导入。复杂文档解析和网页爬取作为后续扩展。

## 7. 开发与部署技术

| 类型 | 选型 | 用途 |
|---|---|---|
| 接口调试 | Apifox / Postman | 调试云端接口 |
| 数据库管理 | pgAdmin / DBeaver | 查看和维护 PostgreSQL 数据 |
| 版本管理 | Git | 管理代码和文档 |
| 演示部署 | 单机部署 | 降低课程答辩演示复杂度 |

## 8. 技术栈边界

第一版不采用以下技术：

1. Elasticsearch。
2. Milvus 或其他独立向量数据库。
3. 微服务架构。
4. Kubernetes。
5. 移动端原生开发。
6. 复杂分布式爬虫。

这些能力超出第一版课程项目范围，后续可以在核心功能稳定后逐步扩展。
