# 校园生活百事通用例规约

## 1. 文档说明

本文档用于描述校园生活百事通系统的主要用例，明确参与者、前置条件、主成功场景、异常流程和后置条件。本文档与《软件需求规约》和《项目规划》配套使用，可作为后续系统设计、测试用例编写和答辩说明的依据。

## 2. 参与者

| 参与者 | 说明 |
|---|---|
| 学生用户 | 注册登录、查询校园生活信息、提交新的问答内容、查看自己的投稿历史 |
| 管理员 | 通过 `admin-console` 负责知识库维护、批量导入数据和审核用户投稿 |
| 系统服务 | 负责检索问答、意图识别、日志记录和热点统计 |
| 大模型服务 | 可选外部服务，用于根据检索结果生成更自然的回答 |

## 4. 用例详细规约

### 4.1 UC-01 学生自然语言提问

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-01 |
| 用例名称 | 学生自然语言提问 |
| 主要参与者 | 学生用户 |
| 相关参与者 | 系统服务、大模型服务 |
| 优先级 | 高 |
| 触发条件 | 学生在问答页面输入问题并提交 |

前置条件：

1. 客户端可以连接云端服务。
2. 知识库中存在至少一条已审核通过的知识。

主成功场景：

1. 学生打开问答页面。
2. 学生输入问题，例如“食堂几点关门？”。
3. 客户端将问题提交到云端问答接口。
4. 系统对问题进行清洗和意图识别。
5. 系统执行关键词检索和语义检索。
6. 系统对候选结果进行混合排序。
7. 系统返回最匹配答案、匹配问题、置信度和相关问题。
8. 客户端展示回答内容。
9. 系统记录本次查询日志。

异常流程：

1. 如果问题为空，客户端提示用户输入问题。
2. 如果云端服务不可用，客户端提示连接失败。
3. 如果未找到高置信度答案，系统提示暂未找到高度匹配的信息，并展示相关问题或引导用户提交新问答。

后置条件：

1. 用户看到答案或未命中提示。
2. 系统保存查询日志。

### 4.2 UC-02 查看热点问题

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-02 |
| 用例名称 | 查看热点问题 |
| 主要参与者 | 学生用户 |
| 相关参与者 | 系统服务 |
| 优先级 | 中 |
| 触发条件 | 学生进入热点问题页面 |

前置条件：

1. 系统中存在查询日志。
2. 云端服务可用。

主成功场景：

1. 学生进入热点问题页面。
2. 客户端请求热点问题接口。
3. 系统按查询次数统计高频问题。
4. 系统返回热点问题列表。
5. 客户端展示问题名称、查询次数和相关分类。
6. 学生点击某个热点问题。
7. 客户端进入问答流程或展示对应答案。

异常流程：

1. 如果暂无查询日志，客户端展示空状态。
2. 如果请求失败，客户端提示加载失败。

后置条件：

1. 学生看到热点问题列表或空状态。

### 4.3 UC-03 用户提交新问答

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-03 |
| 用例名称 | 用户提交新问答 |
| 主要参与者 | 学生用户 |
| 相关参与者 | 管理员 |
| 优先级 | 高 |
| 触发条件 | 学生在贡献页面提交新问题和参考答案 |

前置条件：

1. 客户端可以连接云端服务。
2. 学生用户已登录。

主成功场景：

1. 学生打开用户贡献页面。
2. 学生填写问题、参考答案、分类、标签、信息来源和备注。
3. 学生提交表单。
4. 客户端校验必填字段。
5. 云端校验学生登录状态。
6. 云端保存投稿内容，状态设置为 `pending`，并绑定当前学生用户。
7. 客户端提示提交成功，等待管理员审核。

异常流程：

1. 如果问题或答案为空，客户端提示补全必填字段。
2. 如果内容长度超过限制，客户端提示修改内容。
3. 如果学生未登录，客户端提示先登录。
4. 如果提交失败，客户端提示稍后重试。

后置条件：

1. 投稿内容进入待审核列表。
2. 投稿内容出现在该学生个人中心的投稿历史中。
3. 未审核内容不会进入正式知识库。

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-15 |
| 用例名称 | 学生注册 |
| 主要参与者 | 学生用户 |
| 相关参与者 | 系统服务 |
| 优先级 | 高 |
| 触发条件 | 学生打开注册页面并提交注册信息 |

前置条件：

1. 客户端可以连接云端服务。

主成功场景：

1. 学生打开注册页面。
2. 学生填写账号、密码和昵称。
3. 客户端提交注册请求。
4. 系统校验账号是否可用。
5. 系统保存学生账户和密码哈希。
6. 客户端提示注册成功。

异常流程：

1. 如果账号或密码为空，客户端提示补全信息。
2. 如果账号已存在，系统提示账号不可用。
3. 如果密码不符合要求，客户端提示修改密码。

后置条件：

1. 注册成功后，学生可以使用账号和密码登录。

### 4.7 UC-04 管理员登录

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-04 |
| 用例名称 | 管理员登录 |
| 主要参与者 | 管理员 |
| 相关参与者 | 系统服务 |
| 优先级 | 高 |
| 触发条件 | 管理员访问管理功能 |

前置条件：

1. 系统中已存在管理员账户。
2. 云端服务可用。

主成功场景：

1. 管理员打开登录页面。
2. 管理员输入账号和密码。
3. `admin-console` 提交登录请求。
4. 系统校验账号和密码。
5. 系统生成并返回 JWT。
6. `admin-console` 保存登录状态。
7. 管理员进入 `admin-console` 管理页面。

异常流程：

1. 如果账号或密码为空，`admin-console` 提示补全信息。
2. 如果账号或密码错误，系统返回登录失败。
3. 如果 JWT 过期，系统要求管理员重新登录。

后置条件：

1. 登录成功后，管理员可以访问管理接口。
2. 登录失败时，不允许访问管理功能。

### 4.8 UC-05 新增知识条目

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-05 |
| 用例名称 | 新增知识条目 |
| 主要参与者 | 管理员 |
| 相关参与者 | 系统服务 |
| 优先级 | 高 |
| 触发条件 | 管理员在 `admin-console` 知识库管理页面点击新增 |

前置条件：

1. 管理员已登录。

主成功场景：

1. 管理员打开 `admin-console` 知识库管理页面。
2. 管理员点击新增知识。
3. 管理员填写标题、问题、答案、分类、标签和状态。
4. `admin-console` 提交新增请求。
5. 系统校验必填字段。
6. 系统保存知识条目。
7. 系统更新检索内容。
8. `admin-console` 提示新增成功。

1. 如果必填字段缺失，系统返回校验错误。
2. 如果管理员登录状态失效，系统拒绝请求并要求重新登录。
3. 如果检索内容更新失败，系统保存知识条目并标记为待更新。

后置条件：

1. 新知识条目出现在知识库列表中。
2. 如果状态为 `approved`，该知识可被学生检索。

| 主要参与者 | 管理员 |
| 相关参与者 | 系统服务 |
| 优先级 | 高 |
| 触发条件 | 管理员选择已有知识条目并点击编辑 |

3. 管理员修改标题、问题、答案、分类、标签或状态。
4. `admin-console` 提交修改请求。

2. 检索结果使用更新后的内容。

### 4.10 UC-07 删除知识条目

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-07 |
| 用例名称 | 删除知识条目 |
| 主要参与者 | 管理员 |
| 相关参与者 | 系统服务 |
| 优先级 | 中 |
| 触发条件 | 管理员在 `admin-console` 知识库列表中点击删除 |

前置条件：

1. 管理员已登录。
2. 知识条目存在。

主成功场景：

1. 管理员选择要删除的知识条目。
2. `admin-console` 弹出确认提示。
3. 管理员确认删除。
4. 系统删除知识条目或将其标记为不可用。
5. 系统同步删除或禁用关联检索内容。

异常流程：

1. 如果管理员取消确认，则不执行删除。
2. 如果知识条目不存在，系统提示记录不存在。
3. 如果管理员登录状态失效，系统拒绝请求。

后置条件：

1. 被删除或禁用的知识不再出现在学生检索结果中。

### 4.12 UC-09 批量导入常见问答

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-09 |
| 用例名称 | 批量导入常见问答 |
| 主要参与者 | 管理员 |
| 相关参与者 | 系统服务 |
| 优先级 | 高 |
| 触发条件 | 管理员上传 CSV 或 Excel 文件 |

前置条件：

1. 管理员已登录。
2. 上传文件格式符合系统要求。

主成功场景：

1. 管理员进入 `admin-console` 知识库管理页面。
2. 管理员选择批量导入。
3. 管理员上传 CSV 或 Excel 文件。
4. 系统校验文件类型和大小。
5. 系统解析文件内容。
6. 系统校验每行数据的必填字段。
7. 系统批量保存合法数据。
8. 系统为导入数据更新检索内容。
9. `admin-console` 展示导入成功数量和失败数量。

异常流程：

1. 如果文件类型不支持，系统拒绝导入。
2. 如果文件内容为空，系统提示无可导入数据。
3. 如果部分行校验失败，系统导入合法数据并返回失败原因。
4. 如果全部行失败，系统不写入数据并返回错误详情。

后置条件：

1. 合法数据进入知识库。
2. 导入结果可供管理员查看。

### 4.13 UC-10 审核用户投稿

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-10 |
| 用例名称 | 审核用户投稿 |
| 主要参与者 | 管理员 |
| 相关参与者 | 学生用户、系统服务 |
| 优先级 | 高 |
| 触发条件 | 管理员进入 `admin-console` 投稿审核页面 |

前置条件：

1. 管理员已登录。
2. 系统中存在待审核投稿。

主成功场景：

1. 管理员打开 `admin-console` 投稿审核页面。
2. 系统展示待审核投稿列表。
3. 管理员查看投稿详情。
4. 管理员选择审核通过。
5. 系统将投稿状态更新为 `approved`。
6. 系统将投稿内容写入正式知识库。
7. 系统更新检索内容。
8. `admin-console` 提示审核成功。

扩展流程：

1. 管理员可以选择驳回投稿。
2. 管理员填写审核备注。
3. 系统将投稿状态更新为 `rejected`。
4. 该投稿不会进入正式知识库。

异常流程：

1. 如果投稿不存在，系统提示记录不存在。
2. 如果管理员登录状态失效，系统拒绝审核请求。
3. 如果写入知识库失败，系统保留投稿待审核状态并提示失败原因。

后置条件：

1. 审核通过的投稿进入知识库。
2. 审核驳回的投稿不会被学生检索到。

### 4.14 UC-11 系统记录查询日志

| 用例名称 | 系统记录查询日志 |

| 相关参与者 | 学生用户 |
| 优先级 | 高 |
| 触发条件 | 学生提交问答请求 |

| 相关参与者 | 学生用户 |
| 优先级 | 中 |
| 触发条件 | 客户端请求热点问题列表 |

前置条件：

1. 系统中存在查询日志。

主成功场景：

1. 系统读取查询日志。
2. 系统按归一化问题或命中知识条目聚合查询次数。
3. 系统按查询次数降序排序。
4. 系统过滤不可公开的知识内容。
5. 系统返回排行榜数据。

异常流程：

1. 如果暂无数据，系统返回空列表。
2. 如果统计失败，系统返回错误信息。

1. 客户端可展示热点问题排行榜。

### 4.16 UC-13 系统执行智能检索

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-13 |
| 用例名称 | 系统执行智能检索 |
| 主要参与者 | 系统服务 |
| 相关参与者 | 学生用户 |
| 优先级 | 高 |
| 触发条件 | 问答接口收到用户问题 |

前置条件：

1. 知识库中存在 `approved` 状态的知识。
2. 检索模块可用。

主成功场景：

1. 系统清洗用户问题。
2. 系统提取关键词。
3. 系统根据规则识别意图。
4. 系统执行关键词检索。
5. 系统执行语义检索。
6. 系统合并关键词检索和语义检索结果。
7. 系统合并候选结果。
8. 系统按关键词匹配、语义相似度和意图匹配计算最终分数。
9. 系统返回排序后的候选结果。

异常流程：

1. 如果语义检索不可用，系统降级为关键词检索。
2. 如果关键词检索无结果，系统仍尝试语义检索。
3. 如果所有检索均无结果，系统返回未命中状态。

后置条件：

1. 问答模块获得可用于返回答案的候选结果。

### 4.17 UC-14 大模型增强回答

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-14 |
| 用例名称 | 大模型增强回答 |
| 主要参与者 | 大模型服务 |
| 相关参与者 | 系统服务、学生用户 |
| 优先级 | 低 |
| 触发条件 | 系统配置启用大模型增强，且检索到有效知识内容 |

前置条件：

1. 大模型服务配置可用。
2. 系统已经检索到相关知识内容。

主成功场景：

1. 系统将用户问题和检索结果组装为提示词。
2. 系统要求大模型只能基于给定知识回答。
3. 大模型返回自然语言回答。
4. 系统将大模型回答和引用的知识条目返回给客户端。
5. 客户端展示更自然的回答。

异常流程：

1. 如果大模型服务调用失败，系统降级返回原始知识库答案。
2. 如果检索结果为空，系统不调用大模型。
3. 如果大模型返回内容为空，系统降级返回原始知识库答案。

后置条件：

1. 学生看到自然语言回答或原始知识库答案。
2. 基础问答能力不依赖大模型可用性。
