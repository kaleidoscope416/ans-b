# 校园生活百事通架构设计文档

## 1. 文档目的

本文档描述校园生活百事通系统的总体架构、分层结构、模块边界、核心流程、部署方式和关键架构决策。接口细节见《接口文档》，数据库表结构见《数据库设计文档》。

## 2. 架构目标

系统架构需要满足：

1. 支持学生自然语言问答。
2. 支持学生注册、登录和个人中心。
3. 支持管理员维护知识库。
4. 支持用户投稿和审核入库。
5. 支持热点问题统计。
6. 支持智能检索能力扩展。
7. 保持第一版部署和演示简单。

## 3. 总体架构

```text
Wails 学生桌面客户端
  |
  | HTTP / JSON
  v
Go 云端服务
  ^
  | HTTP / JSON
  |
admin-console 管理端

Go 云端服务
  |
  | SQL / Vector Query
  v
PostgreSQL + pgvector

可选外部服务：
  - embedding 模型服务
  - 大模型 API
```

学生客户端负责学生问答、热点问题、注册登录、个人中心和投稿。`admin-console` 负责管理员登录、知识库维护、FAQ 导入和投稿审核。云端服务负责业务逻辑和智能检索，数据库负责业务数据和向量数据持久化。

## 4. 架构分层

### 4.1 表现层

表现层由学生桌面客户端和 `admin-console` 共同承担。

职责：

1. 学生桌面客户端展示问答、热点问题、注册登录、个人中心和投稿页面。
2. `admin-console` 展示管理员登录、知识库管理和投稿审核页面。
3. 两个前端都通过 HTTP 调用云端接口。
4. 两个前端都需要展示加载、成功、失败和空状态。

### 4.2 接口层

接口层由云端服务提供。

职责：

1. 接收客户端请求。
2. 校验请求参数。
3. 执行管理员鉴权。
4. 调用业务模块。
5. 返回统一响应。

### 4.3 业务层

业务层封装系统业务规则。

职责：

1. 学生注册和登录。
2. 管理员登录。
3. 知识库维护。
4. 问答流程编排。
5. 投稿和审核。
6. 查询日志和热点统计。

### 4.4 检索层

检索层封装智能问答检索能力。

职责：

1. 问题清洗。
2. 意图识别。
3. 关键词检索。
4. 语义检索。
5. 候选结果排序。
6. 低置信度处理。

### 4.5 数据访问层

数据访问层封装数据库读写。

职责：

1. 业务数据读写。
2. 检索数据读写。
3. 查询日志读写。
4. 事务处理。

## 5. 后端模块划分

| 模块 | 职责 |
|---|---|
| `auth` | 学生登录、管理员登录、密码校验、令牌生成和鉴权 |
| `user` | 学生注册、账户信息和个人中心 |
| `knowledge` | 知识新增、编辑、删除、查询和导入 |
| `qa` | 问答请求编排、答案返回、未命中处理 |
| `search` | 问题预处理、意图识别、检索和排序 |
| `submission` | 用户投稿、投稿查询、审核通过和驳回 |
| `analytics` | 查询日志记录、热点问题统计 |
| `model` | embedding 和大模型调用封装 |
| `storage` | 上传文件和导入文件存储 |

## 6. 前端模块划分

### 6.1 学生客户端模块

| 模块 | 职责 |
|---|---|
| 问答页面 | 输入问题、展示答案和相关问题 |
| 热点页面 | 展示热点问题排行榜 |
| 注册登录页面 | 学生注册和登录 |
| 个人中心页面 | 展示账户信息和投稿历史 |
| 投稿页面 | 提交新的问答内容 |
| API 模块 | 封装学生端接口调用 |
| 状态模块 | 管理问答结果和加载状态 |

### 6.2 admin-console 模块

| 模块 | 职责 |
|---|---|
| 登录页面 | 管理员账号密码登录 |
| 知识库管理页面 | 查询、新增、编辑、删除知识 |
| FAQ 导入页面 | 上传并导入 FAQ 文件 |
| 投稿审核页面 | 查看、通过和驳回用户投稿 |
| API 模块 | 封装管理员接口调用 |
| 状态模块 | 管理管理员登录状态和管理端页面状态 |

## 7. 核心流程

### 7.1 学生问答流程

```text
学生输入问题
  ↓
学生客户端提交问题
  ↓
云端校验请求
  ↓
问答模块调用检索模块
  ↓
检索模块返回候选结果
  ↓
问答模块判断置信度
  ↓
系统记录查询日志
  ↓
学生客户端展示答案或未命中提示
```

### 7.2 知识维护流程

```text
管理员在 admin-console 提交知识变更
  ↓
云端校验管理员身份
  ↓
知识模块校验数据
  ↓
系统保存知识内容
  ↓
系统更新检索内容
  ↓
admin-console 提示操作结果
```

### 7.3 投稿审核流程

```text
学生登录
  ↓
学生提交新问答
  ↓
系统保存为待审核
  ↓
管理员在 admin-console 查看投稿
  ↓
管理员审核通过或驳回
  ↓
通过内容进入知识库
```

### 7.4 个人中心流程

```text
学生登录
  ↓
学生进入个人中心
  ↓
系统校验学生身份
  ↓
系统查询该学生的投稿历史
  ↓
学生客户端展示投稿状态和审核备注
```

### 7.5 热点统计流程

```text
用户提问
  ↓
系统记录查询日志
  ↓
系统聚合高频问题
  ↓
学生客户端展示热点排行榜
```

## 8. 安全架构

1. 普通问答和热点问题可开放访问。
2. 学生投稿和个人中心必须校验学生身份。
3. 管理功能必须校验管理员身份。
4. 学生和管理员密码必须哈希存储。
5. 文件导入必须校验类型和大小。
6. 用户投稿必须审核后才能公开。
7. 系统不向普通用户暴露内部错误细节。

## 9. 部署架构

第一版推荐单机或单服务器部署。

```text
学生电脑
  |
  | Wails 学生桌面客户端
  v
服务器或本机演示环境
  |
  |-- admin-console
  |-- Go 云端服务
  |-- PostgreSQL + pgvector
  |-- 本地文件目录
```

演示时可以将学生客户端、`admin-console`、云端服务和数据库部署在同一台机器。实际使用时，学生客户端安装在用户电脑，`admin-console` 和云端服务部署在服务器或内网环境。

## 10. 扩展设计

### 10.1 文档导入扩展

后续可新增文档解析模块，支持 PDF、Word、Markdown、TXT 和网页内容导入。

### 10.2 检索能力扩展

后续可增强中文分词、同义词词典、embedding 模型、重排序模型和独立搜索引擎。

### 10.3 大模型能力扩展

后续可支持多模型供应商、流式回答、引用来源展示和多轮问答上下文。

## 11. 关键架构决策

### 11.1 采用客户端与云端分离

知识库集中维护，学生客户端和 `admin-console` 都保持轻量，后续可复用云端服务扩展 Web 端或移动端。

### 11.2 第一版采用模块化单体

课程项目规模有限，模块化单体比微服务更容易开发、部署和演示，同时仍能保持清晰模块边界。

### 11.3 大模型作为可选增强

基础问答依赖知识库检索，大模型只负责优化回答表达，避免外部服务不可用影响核心功能。

## 12. 架构边界

第一版不覆盖：

1. 多学校多租户。
2. 大规模分布式部署。
3. 自动爬取公众号。
4. 移动端应用。
5. 人工客服系统。
